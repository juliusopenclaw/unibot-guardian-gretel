# UniBot Data Protection Screening

Status: public-safe planning draft, not legal advice and not Datenschutz
approval.

The data-protection screening is generated by `unibot.privacy` and exposed
through:

- `POST /api/unibot/data-protection-screening`
- `POST /api/unibot/data-protection-screening-markdown`

## Purpose

The screening gives Datenschutz reviewers a structured first view of UniBot's
practice MVP and synthetic pilot plan. It lists processing principles,
processing activities, risks, controls, open decisions, review gates, and source
cards.

## Covered Activities

- Prompt Card generation.
- External KI output postfilter.
- Local Help Ledger.
- Demo feedback and manual GitHub issue drafts.
- Synthetic pilot records.
- Future `exam_controlled` mode as blocked until authority review.

## Public Boundary

The public draft does not approve a real pilot, cloud storage, exam use,
official assessment, disciplinary KI-detection, or handling of medical or
accommodation personal data. It keeps those decisions open for Datenschutz,
ethics or supervisor review, Pruefungsamt, IT/SZI, and teaching review.

## Open Decisions

- Approved storage location for any real pilot records.
- Retention before and after aggregation.
- Access rights for non-public records.
- Whether a formal Datenschutz-Folgenabschaetzung is required.
- Whether any cloud service may be used.
- Withdrawal and deletion workflow before recruitment.

## Release Rule

The screening is public-draft ready only when the public-safety scan passes,
Datenschutz review is still explicitly required before a real pilot, and
`exam_deployment_status` remains `not_cleared`.
