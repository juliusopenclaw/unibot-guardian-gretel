# UniBot Compliance Matrix

Status: public-safe authority-review draft, not legal advice and not exam
clearance.

The compliance matrix is generated by `unibot.compliance` and exposed through:

- `POST /api/unibot/compliance-matrix`
- `POST /api/unibot/compliance-matrix-markdown`

## Purpose

The matrix links UniBot's source cards to concrete product rules, implemented
controls, blocked uses, and verification evidence. It is meant to make review
by Pruefungsamt, Inklusionsbuero, Datenschutz, IT/SZI, teaching staff, and
thesis supervision easier and more transparent.

## Requirement Groups

- `exam-clearance-boundary`: exam-controlled use remains blocked until written
  authority clearance exists.
- `accessibility-neutrality`: accessibility support is documented separately
  and never lowers the private formative score.
- `data-minimisation-and-public-safety`: only minimal local metadata is stored,
  and public artifacts must not contain private material.
- `ai-act-high-risk-avoidance`: UniBot avoids official assessment,
  proctoring, and disciplinary KI-detection use.
- `research-integrity-and-reproducibility`: sources, methods, versions,
  limitations, and reproducible checks are documented.
- `socratic-pedagogy-and-learning-evidence`: external AI help is transformed
  into next-step scaffolding rather than final answers.
- `colab-gemini-output-validation`: Colab/Gemini output is validated through
  visible Prompt Card and Postfilter flows.
- `browser-overlay-limit`: a normal browser overlay is practice UX, not hard
  exam security.
- `controlled-notebook-route`: a future exam variant should prefer a controlled
  local or managed notebook channel over a generic browser overlay.

## Release Rule

The matrix is considered ready only when all referenced source cards exist, the
public-safety scan passes, and `exam_deployment_status` remains `not_cleared`.
That means public draft review is possible, but exam use still needs written
university review and approval.
