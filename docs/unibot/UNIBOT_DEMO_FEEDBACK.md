# UniBot Demo Feedback

Status: local public-safe bug-report contract.

The demo feedback layer is implemented in `unibot.feedback` and exposed through:

- `POST /api/unibot/demo-feedback/template`
- `POST /api/unibot/demo-feedback/validate`
- `POST /api/unibot/demo-feedback/append`
- `POST /api/unibot/demo-feedback/list`
- `POST /api/unibot/demo-feedback/summary`
- `POST /api/unibot/demo-feedback/public-summary`

The browser side panel also exposes a compact Demo-Feedback form. It can
validate feedback through the local API and store validated feedback locally.
If the API is offline, validation falls back to a local public-safety check, but
storage stays unavailable.

## Purpose

After a local demo, the tester should be able to report what failed without
accidentally publishing private data or raw course material. The feedback layer
turns a loose bug report into a structured, reproducible, local-only record.

## Required Fields

- `scenario_id`
- `what_i_tried`
- `expected`
- `what_happened`
- `button_or_endpoint`
- `private_data_removed`

## Review-Board Claim Alignment

Validated feedback records include `claim_alignment` with the contract
`unibot-demo-feedback-release-review-board-claim-alignment-v1`. This keeps
local demo feedback aligned with downstream feedback triage, GitHub issue
drafts, the publication package, release runbook, review-board thesis/evaluation
boundary, and public-safety gates.

The contract is local-only and public-summary-only. It does not publish raw
feedback, create issues, approve publication, recruit participants, call
providers, clear exam use, or approve a thesis submission.
Unsafe or incomplete feedback is blocked and not stored. Public summaries
exclude screenshots, copied free text, local paths, emails, health or
accommodation details, real exam work, and raw private course text.

## Public Summary

The public summary contains only:

- number of reports
- reports needing follow-up
- counts by scenario
- counts by outcome
- counts by severity

Follow-up maintenance triage is generated by:

- `POST /api/unibot/demo-feedback/triage`
- `POST /api/unibot/demo-feedback/triage-markdown`
