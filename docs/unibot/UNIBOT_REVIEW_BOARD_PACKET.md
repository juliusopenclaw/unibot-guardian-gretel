# UniBot Review Board Packet

Status: separate institutional review artifact (not approval).

This packet is generated by `unibot.review_board` and exposed through:

- `POST /api/unibot/review-board-packet`
- `POST /api/unibot/review-board-packet-markdown`

What it does:

- Collect reviewer packets for Pruefungsamt, Inklusionsbuero, Datenschutz,
  IT/SZI, teaching, and thesis supervision.
- Record open institutional decision points before moving beyond local practice.
- Define cross-cutting red lines that remain active in all modes.
- Summarise evidence status from handoff, compliance, pilot, and data-protection
  layers.
- Keep language non-committal (`beantragt / nicht offiziell freigegeben`) until
  written approvals exist.

Current status in draft:

- `status`: `draft_for_institutional_review`
- `schema`: `unibot-review-board-packet-v1`
- `exam_deployment_status`: `not_cleared`

Reviewer packet content:

- `reviewer`: role name
- `mandate`: what the reviewer should decide
- `decisions_needed`: open decisions required by that reviewer
- `required_evidence`: artifacts and evidence types expected
- `required_source_cards`: source-card list with authority context
- `must_not_claim`: claims that must be avoided in public language
- `clearance_text_must_include`: minimum wording for transparent communication

Readiness for publication is a separate check:

- `reviewer_packets` count >= 6
- each reviewer has at least one open decision
- `public_safety_status` pass
- `exam_deployment_status` not_cleared

The packet is intentionally not an approval and does not replace formal
authority decisions. It prepares the right questions in a structured, reproducible,
institutional format.
